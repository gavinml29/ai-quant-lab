# TASK3 — Stock Indicator Lab (指标计算实验室)

## 产品概述

一个基于 Web 的交互式技术指标计算工具。用户可选择任意 A 股标的，对 RSI / MACD / BOLL / ATR 四个指标分别调节参数并实时重绘，所见即所得。

**核心价值**：把传统 Jupyter Notebook 中的"改参数 → 重跑 → 等结果"循环，压缩为拖动滑块即刻生效的交互体验。

---

## 用户故事

1. 作为分析者，我想**切换不同股票**查看指标，而不是固定一只。
2. 作为研究者，我想**拖动滑块调节 RSI 周期**，立即看到曲线变化。
3. 作为交易者，我想**对比不同参数的 MACD** 金叉/死叉时点的差异。
4. 作为学习者，我想**理解公式**，所以每个指标面板都要展示计算逻辑。

## 数据模型

### 输入

| 字段 | 来源 | 说明 |
|------|------|------|
| 股票代码 | 用户选择 / 搜索 | A 股代码（如 `sh600703`） |
| 日期范围 | 日期选择器 | 默认近 3 年 |
| RSI 参数 N | 滑块 | 默认 14, 范围 2~100 |
| MACD 参数 (fast, slow, signal) | 三个滑块 | 默认 12/26/9 |
| BOLL 参数 (N, K) | 两个滑块 | 默认 20/2 |
| ATR 参数 N | 滑块 | 默认 14, 范围 2~100 |

### 计算流程

```
stock_code, date_range
        │
        ▼
   ┌──────────┐
   │ 数据获取   │ ← 需对接数据接口 (westock-data / tushare)
   └────┬─────┘
        │ OHLCV (date, open, high, low, close)
        ▼
   ┌──────────┐
   │ 参数解耦   │ ← 每个指标独立接收 OHLCV + 自己的参数
   └────┬─────┘
        │
   ┌────┼────┬────┬────┐
   ▼    ▼    ▼    ▼    ▼
  RSI  MACD BOLL ATR  ...
   │     │     │     │
   ▼     ▼     ▼     ▼
  Chart Chart Chart Chart
```

### 每个指标的输出

```
{
  indicator: "RSI",
  params: { N: 14 },
  data: [
    { date: "2024-01-02", value: 55.3 },
    ...
  ],
  signals: [
    { date: "2024-03-15", type: "oversold", value: 28.5 },
    ...
  ]
}
```

## 交互设计

### 全局区

- **股票搜索框**：支持代码（600703）或名称（三安光电）模糊搜索
- **日期范围**：两个日期输入框（开始 / 结束）
- **加载按钮**：触发数据拉取 + 四项指标重新计算 + 全部图表重绘
- **状态指示**：加载中的 spinner / 数据行数显示

### 参数控制（每个面板）

- **水平滑块**：拖动过程**实时重绘**对应图表（防抖 300ms）
- **数值输入框**：滑块旁边显示当前值，可直接输入
- **重置按钮**：恢复默认参数
- **公式折叠区**：点击展开 LaTeX 公式

### 图表交互

- **缩放**：鼠标滚轮 / 双指缩放时间轴
- **平移**：拖拽移动可视范围
- **十字光标**：hover 显示精确数值
- **信号标注**：超买/超卖、金叉/死叉、突破等关键点高亮

### 多面板联动

- 所有图表共享同一个**时间轴**（同步缩放/平移）
- 图表高度可拖拽调节

## 技术方案

### 选型

| 层 | 技术 | 理由 |
|----|------|------|
| 框架 | 纯 HTML/CSS/JS | 零依赖、轻量、可直接双击打开 |
| 图表库 | **Chart.js** + chartjs-plugin-zoom | 轻量(60KB)、Canvas 渲染快、中文友好 |
| 数据源 | westock-data CLI → JSON 文件 或 内嵌 CSV | 离线模式可用预加载数据 |
| 计算 | 纯 JavaScript (手写，同 notebook 逻辑) | 与 Python 版本公式严格一致 |
| 样式 | Tailwind CSS (CDN) | 快速原型、响应式 |

### 为什么选 Chart.js 而不是 Plotly/ECharts

- Chart.js 体积最小 (60KB vs Plotly 3MB vs ECharts 1MB)
- Canvas 渲染比 SVG 更适合金融时序数据（大盘数据不卡）
- chartjs-plugin-zoom 提供开箱即用的缩放/平移
- 不需要 npm/webpack，单文件即可

### 计算层隔离

```
data-layer.js      ← 数据获取、缓存
  ↓
compute-engine.js  ← RSI/MACD/BOLL/ATR 纯函数
  ↓
indicator-chart.js ← 每个指标的 Chart.js 封装
  ↓
app.js             ← 全局状态管理、事件绑定
```

### 每个指标的 compute 函数签名

```javascript
// 返回：{ dates, values, signals }
function computeRSI(closePrices, N = 14) { ... }
function computeMACD(closePrices, fast=12, slow=26, signal=9) { ... }
function computeBOLL(closePrices, N=20, K=2) { ... }
function computeATR(highs, lows, closes, N=14) { ... }
```
